# shokz-diary

每日复盘主流程。通过纯对话方式收集学员输入，自动生成结构化日报。

## 交互模式

**纯对话模式**：不使用 AskUserQuestion。直接输出问题文本，学员在下一条消息中自然回复。这样交互更流畅自然。

## 前置检查

1. 读取 `.shokz/profile.json`
2. 若文件不存在，输出提示并终止：
   ```
   尚未初始化学员配置。请先运行 /shokz-diary:shokz-init
   ```
3. 从 profile 中读取 `outputDir`（若无则默认 `.shokz/reports/`）
4. 若当天 `{outputDir}/{YYYY-MM-DD}.md` 已存在，告知学员并询问是否覆盖
5. **记录开始时间**（ISO 8601 格式，精确到分钟）

## 执行流程

### Phase 1: 交互式提问（目标：2 轮收集完毕）

**第 1 轮 — 一次性输出三个问题：**

> 今天的复盘开始，回答以下三个问题就行：
>
> 1. **今天做了什么？** 流水账就行，3-5 句话。
> 2. **哪里卡住了 / 感觉不太对？** 没有的话写"无"。
> 3. **想跟导师讨论什么？** 没有的话写"无"。

等待学员回复。

**第 2 轮 — 深度追问（核心环节）：**

这是 Agent 作为"半个导师"的关键环节。不是问"能不能再说具体点"，而是基于学员的回答 + 实习目标 + 专业判断，生成**导师级的深度问题**。

**追问生成逻辑**：
1. 结合 profile.goal 理解学员当前阶段和目标
2. 分析学员回答中隐含的决策点、风险点、认知盲区
3. 生成 1-3 个有针对性的深度问题

**追问应该是什么样的**：
- 指向决策依据："你们讨论定位时，是基于什么数据/假设得出这个方向的？"
- 指向知识缺口："华强北看到的价格带分布，对你们的定价策略有什么影响？"
- 指向下一步行动的合理性："调研和讨论之间有没有形成闭环？调研结论会怎么反馈到产品决策里？"
- 指向风险识别："如果 C 端定位走这个方向，最大的不确定性是什么？"

**追问不应该是什么样的**：
- ❌ "能再具体一点吗？"（空洞）
- ❌ "开了什么会？"（信息搬运）
- ❌ "还有什么想补充的吗？"（形式化）

**追问数量**：1-3 个问题，合并在一条消息中输出。

**如果学员回答已经足够有深度**（包含了决策依据、具体产出、下一步计划），可以跳过追问直接进入 Phase 2。

**学员回复后**：接受回答，**记录结束时间**，不再二次追问。直接进入 Phase 2。

### Phase 2: 生成日报

三个问题收集完毕后，使用 `diary-generation` skill 的规则生成输出区：

**输入：**
- 学员 profile（name, mentor, goal）
- 当前日期
- Phase 1 收集的三个问题的回答

**生成内容（输出区）：**
- A. 差距识别（对照实习目标，最多 3 项）
- B. 思路拓展（1-2 个具体可操作的建议）
- C. 明日行动方法（最多 2 项，每项含「怎么做」和「完成标志」）
- D. 导师沟通要点（若学员无问题则写"本日无需导师沟通"）

### Phase 3: 组装并输出

组装完整日报，格式如下：

```markdown
# 每日复盘 — {日期}

| 姓名 | 导师 | 日期 | 用时 |
|------|------|------|------|
| {name} | {mentor} | {YYYY-MM-DD} | {结束时间 - 开始时间} 分钟 |

**实习目标**：{goal}

---
```

然后：
1. 确保 `{outputDir}` 目录存在（从 profile.outputDir 读取，默认 `.shokz/reports/`）
2. 将日报写入 `{outputDir}/{YYYY-MM-DD}.md`
3. 在终端完整展示生成的日报
4. 输出：`日报已保存至 {outputDir}/{YYYY-MM-DD}.md`

## 关键约束

- 不使用 AskUserQuestion，所有交互通过纯文本对话完成
- 每次只输出一个问题，等待回复
- 整个流程目标 ≤ 15 分钟
- 语气直接专业，不客套不鼓励
