# shokz-init

初始化学员的每日复盘配置。

## 交互模式

**纯对话模式**：不使用 AskUserQuestion。直接输出问题，学员自然回复。

## 执行流程

### 1. 检查现有配置

读取当前工作目录下的 `.shokz/profile.json`。

- **若已存在**：展示当前配置内容，问学员"要覆盖重新设置吗？"。若学员说不需要，结束执行。
- **若不存在**：继续下一步。

### 2. 收集学员信息

**一次性**输出以下问题（init 场景信息明确，不需要逐个问）：

> 请告诉我以下信息：
> 1. 你的姓名
> 2. 导师姓名
> 3. 入职日期（如 2026-07-01）
> 4. 核心实习目标（与导师对齐后的那个目标）
> 5. 日报输出位置（文件夹路径，默认为当前目录下的 .shokz/reports/）
>
> 目标示例：在 3 个月内，能够独立完成一份达到团队交付标准的行业研究报告。

等学员回复后，如果目标过于模糊（如"学习产品""做项目"），追问一次：
> 能否更具体一些？比如：具体要交付什么成果？达到什么标准？在多长时间内？

如果学员未指定输出位置，使用默认值 `.shokz/reports/`。

### 3. 写入配置

创建目录结构并写入配置文件：

```bash
mkdir -p .shokz/reports
```

`profile.json` 格式：
```json
{
  "name": "学员姓名",
  "mentor": "导师姓名",
  "goal": "核心实习目标",
  "startDate": "YYYY-MM-DD",
  "outputDir": "日报输出路径（绝对路径或相对路径）",
  "createdAt": "ISO 8601 时间戳"
}
```

### 4. 确认完成

在终端输出：

```
--- 初始化完成 ---

姓名：{name}
导师：{mentor}
目标：{goal}
入职日期：{startDate}

配置已保存至 .shokz/profile.json
现在可以使用 /shokz-diary:shokz-diary 开始每日复盘。
```

## 注意事项

- 保持对话友好、简洁
- 如果学员对某个字段不确定，允许先填写再后续修改
- 不使用 AskUserQuestion，所有交互通过纯文本完成
