# 日报生成 Skill

本 Skill 定义如何把学员三轮输入转化为导师可提交日报。它只负责 AI 输出区和结构化表达，不改写输入区原话。

## 输入

使用以下信息生成日报：

- 学员稳定配置：`.shokz/profile.json`
- 动态学员画像：`.shokz/learner-profile.md`（若存在）
- 导航索引与未关闭事项：`.shokz/index.md`（若存在）
- 最近 3-5 篇日报与导师反馈（若存在）
- 第 1 轮原话：今天做了什么
- 第 2 轮原话：哪里卡住了 / 感觉不太对
- 第 3 轮原话：有什么想和导师讨论

## 核心原则

- 学员只负责输入区：流水账、卡点现象、导师讨论点。
- 三轮输入是开放承接式访谈，不是机械表单；学员可以提前说到后续话题。
- AI 负责输出区：差距识别、思路拓展、明日行动方法、导师沟通要点。
- 导师主要阅读 AI 输出区和导师反馈区；输入区用于追溯，不用于替代正文。
- 附录必须暴露事实边界和原始表达，避免 AI 把学员表达问题完全美化掉。

## 生成前检查

生成正文前必须先做内部检查，不必单独向学员追问：

1. **输入槽位**：识别三轮中已经出现的事实、卡点、导师讨论点；不要因为学员提前说了后续内容就丢失。
2. **时间归属**：区分 `今日发生`、`跨日补充`、`历史背景`、`导师反馈事件`、`待确认`。
3. **来源归属**：区分 `学生原话`、`历史上下文`、`Agent 推断`、`待确认`。
4. **证据边界**：学生没说、上下文没给、导师没反馈的内容不能写成事实。
5. **沟通风险**：如果 AI 输出明显美化了学员原始表达，必须在附录 A 暴露原始输入和事实边界。

## 来源标记

正文和附录中的关键判断必须标注来源。

来源类型固定为：

- `学生原话`：只来自三轮输入中学员明确说出的内容。
- `历史上下文`：来自 learner-profile、index、最近日报或导师反馈。
- `Agent 推断`：基于输入做出的差距、风险、建议或管理判断。
- `待确认`：证据不足，不能写成事实。

规则：

- 只有学员在本次三轮输入中明确说出的事实，才能标 `学生原话`。
- 历史日报、GitHub 状态、项目状态、画像观察不能标 `学生原话`。
- `Agent 推断` 只能用于差距分析、原因假设、行动建议和沟通问题整理。
- 若无法判断来源，写 `待确认`，不要脑补。

## 时间归属

时间归属固定为：

- `今日发生`
- `跨日补充`
- `历史背景`
- `导师反馈事件`
- `待确认`

规则：

- 第 1 轮中出现“上周五”“昨天”“之前”“第一版”等表达时，不得写入“今日发生”。
- 跨日事件可以进入日报，但必须标为 `跨日补充`、`历史背景` 或 `导师反馈事件`。
- “今日推进”不再作为主标题使用，避免把跨日背景误写成当天工作。

## 日报结构

严格生成以下结构。

### 基本信息

输出表格：

| 字段 | 内容 |
|------|------|
| 姓名 | |
| 导师 | |
| 日期 | |
| 开始时间 | |
| 结束时间 | |
| 耗时 | |

规则：

- 耗时格式为 `{分钟数} 分钟（目标 ≤ 15 分钟）`。
- 若结束时间未知，写 `待确认`。

### 固定锚点：我的实习目标

输出 profile 中的 `goal`。若没有，写 `待确认`。

### 输入区：学员原始输入

完整保留三轮原话，不得摘要替代：

1. 今天做了什么？
2. 今天哪里卡住了 / 感觉不太对？
3. 有什么想跟导师讨论的？

### AI 输出区

#### A. 差距识别

输出表格：

| 差距 | 可能原因 | 优先级 | 依据/来源 |
|------|----------|--------|-----------|

规则：

- 差距必须对照固定实习目标或导师反馈。
- “可能原因”只能是推断，依据/来源必须标 `Agent 推断` 或引用输入证据。
- 如果输入不足，写 `信息不足，待确认`。
- 优先级使用：`高` / `中` / `低`。
- 最多 3 行。

#### B. 思路拓展

输出 1-2 个新的思路或方法。

规则：

- 只针对 A 中的关键差距。
- 每个思路要说明适用场景和为什么能帮助突破卡点。
- 不输出泛泛鼓励语。
- 不替导师做最终决策。

#### C. 明日行动方法

输出表格：

| 行动 | 怎么做 | 完成标志 | 优先级 |
|------|--------|----------|--------|

规则：

- 聚焦 1-2 项。
- `怎么做` 必须是步骤级行动。
- `完成标志` 必须可验证。
- 明日行动必须回应今天的卡点或导师讨论点。
- 不写“继续推进”“多思考”这类不可验收动作。

#### D. 导师沟通要点

输出：

```markdown
**需导师确认：**
1. ...

**需导师提供输入：**
1. ...
```

规则：

- 把学员的疑问整理成导师可回答的问题。
- 在 AI 输出区中，面向导师的问题可优先写成 Yes/No、选择题或具体输入请求；这条规则只适用于生成后的导师沟通要点，不适用于三轮采集阶段。
- 若学员第 3 轮写“无”，但三轮输入中存在定位、标准、优先级、资源依赖、关键决策等风险，可补充“建议确认”，来源标 `Agent 推断`。
- 如果确实无需导师确认，写 `本日暂无需导师确认事项`。

### 导师反馈区

输出表格：

| 反馈项 | 导师意见 |
|--------|----------|
| 对差距分析的补充 | |
| 对行动方法的建议 | |
| 对疑问的回复 | |
| 其他 | |

### 附录 A：事实边界与来源说明

输出表格：

| 信息 | 时间归属 | 来源类型 | 说明 |
|------|----------|----------|------|

规则：

- 必须列出所有可能被误解为“今日发生”的跨日事件。
- 必须列出所有 AI 推断过的关键判断。
- 如果发现学员表达混杂、时间线不清、问题未结构化，要在说明中标出，不要在正文中完全掩盖。

### 附录 B：画像/索引更新建议

默认只生成建议，不写入长期文件。

#### B1. 学员画像更新建议

输出表格：

| 操作 | 观察内容 | 证据来源 | 建议状态 |
|------|----------|----------|----------|

操作使用：

- `新增`
- `更新`
- `关闭`
- `降级`
- `标记过期`
- `暂无`

建议状态使用：

- `观察中`
- `已确认`
- `已修正`
- `已过期`

规则：

- 一次日志中的表现默认只能建议为 `观察中`。
- 只有明确导师反馈或多日重复证据，才能建议 `已确认`。
- 不得把“表达不清”“调研方法不足”写成人格标签。
- 必须写证据来源，如 `2026-07-06 第 1 轮原始输入`、`导师反馈事件`。

#### B2. 导航索引更新建议

输出表格：

| 索引项 | 建议写入内容 |
|--------|--------------|

索引项包括：

- 日期日报
- 项目索引
- 问题类型索引
- 导师反馈索引
- 未关闭事项
- 团队文档沉淀候选

## 质量标准

- 导师 2 分钟内能读完 AI 输出区并判断重点。
- 学员原始输入完整保留，导师需要时可追溯。
- 差距识别必须服务于固定实习目标。
- 明日行动必须可执行、可验收。
- 导师沟通要点必须能直接发给导师。
- 所有建议必须能追溯到输入、历史上下文或导师反馈。

## 禁止行为

- 不要在三轮输入阶段提前分析或追问扩展。
- 不要把三轮输入当成死板表单；用户提前说到的后续内容必须被承接和保留。
- 不要用选择题、判断题或原因分析替学员收束表达。
- 不要把三轮输入压缩成一句话。
- 不要把跨日事件写成今日发生。
- 不要把历史上下文或 Agent 推断标成 `学生原话`。
- 不要虚构结论、数据、产出、GitHub 状态或导师意图。
- 不要把一次表现永久标签化。
- 不要自动写入 `.shokz/learner-profile.md` 或 `.shokz/index.md`；只能给建议，等待主流程获得确认。
