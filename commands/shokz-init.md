# shokz-init

初始化学员每日复盘配置。只把稳定信息写入 `profile.json`，动态评价和成长观察写入独立的本地 Markdown 文件。

## 交互模式

**纯对话模式**：不使用 AskUserQuestion。直接输出问题，等待学员自然回复。

## 设计原则

- `profile.json` 只存稳定配置，不存动态评价。
- `.shokz/learner-profile.md` 存动态学员画像，可持续修正。
- `.shokz/index.md` 存日报导航、问题索引、导师反馈和未关闭事项。
- 初始化只创建空模板，不对学员做任何画像判断。

## 执行流程

### 1. 检查现有配置

读取当前工作目录下的 `.shokz/profile.json`。

- 若已存在：展示当前配置内容，询问“要覆盖重新设置稳定配置吗？”。
- 若学员说不需要：继续检查 `.shokz/learner-profile.md` 和 `.shokz/index.md` 是否存在；缺失则询问是否补建模板。
- 若不存在：继续下一步。

### 2. 收集稳定信息

一次性输出：

> 请告诉我以下信息：
> 1. 你的姓名
> 2. 导师姓名
> 3. 入职日期（如 2026-07-01）
> 4. 核心实习目标（与导师对齐后的那个目标）
> 5. 日报输出位置（文件夹路径，默认为当前目录下的 .shokz/reports/）
>
> 目标示例：在 3 个月内，能够独立完成一份达到团队交付标准的行业研究报告。

如果目标过于模糊（如“学习产品”“做项目”），追问一次：

> 能否更具体一些？比如：具体要交付什么成果？达到什么标准？在多长时间内？

如果学员未指定输出位置，使用默认值 `.shokz/reports/`。

### 3. 创建目录与稳定配置

创建目录：

```bash
mkdir -p .shokz/reports
```

写入 `.shokz/profile.json`：

```json
{
  "name": "学员姓名",
  "mentor": "导师姓名",
  "goal": "核心实习目标",
  "startDate": "YYYY-MM-DD",
  "outputDir": ".shokz/reports/",
  "createdAt": "ISO 8601 时间戳"
}
```

规则：

- 不要向 `profile.json` 添加动态评价、长期 gap、表达习惯、导师偏好等字段。
- 若需要修改稳定配置，后续再次运行 `shokz-init`。

### 6. 确认完成

输出：

```text
--- 初始化完成 ---

姓名：{name}
导师：{mentor}
目标：{goal}
入职日期：{startDate}

稳定配置已保存至 .shokz/profile.json
动态画像模板：.shokz/learner-profile.md
导航索引模板：.shokz/index.md
日报输出目录：{outputDir}

现在可以使用 /shokz-diary:shokz-diary 开始每日复盘。
```

## 注意事项

- 保持对话友好、简洁。
- 如果学员对某个字段不确定，允许先填写，后续再覆盖。
- `.shokz/` 是用户本地数据，不属于插件仓库提交内容。
- 不使用 AskUserQuestion。
