# shokz-calibrate

校准修改当天已生成的日报。学员指出不满意的部分，Agent 重新生成该部分。

## 交互模式

**纯对话模式**：不使用 AskUserQuestion。直接输出问题，学员自然回复。

## 前置检查

1. 获取当前日期（YYYY-MM-DD 格式）
2. 读取 `.shokz/reports/{当天日期}.md`
3. 若文件不存在，输出提示并终止：
   ```
   今天还没有生成日报。请先运行 /shokz-diary:shokz-diary
   ```
4. 读取 `.shokz/profile.json` 获取学员目标（用于重新生成时参考）

## 执行流程

### Step 1: 展示当前日报

在终端展示当天日报的 **AI 输出区**（A/B/C/D 四个部分），不展示输入区和基本信息。

### Step 2: 收集修改意见

直接输出：

> 以上是今天的 AI 输出区。哪部分需要修改？（A/B/C/D，可多选）
> 具体哪里不满意？（比如：太笼统、方向偏了、不符合实际情况、想换个角度）

等待学员回复。学员可以一次性说明要改哪部分 + 怎么改。

### Step 3: 重新生成

基于以下信息重新生成学员指定的部分：
- 原始学员输入（从日报文件的输入区提取）
- 学员 profile 中的目标
- 学员的修改意见

**保留未指定部分不变。**

重新生成时遵循 `diary-generation` skill 中定义的质量标准和格式要求。

### Step 4: 更新文件

1. 将修改后的内容替换到日报文件对应部分
2. 覆盖写入 `.shokz/reports/{当天日期}.md`
3. 在终端展示修改后的完整 AI 输出区
4. 输出：`日报已更新。如仍需调整，可再次运行 /shokz-diary:shokz-calibrate`

## 注意事项

- 每次校准只针对 AI 输出区，不修改学员输入区
- 可以多次运行校准，直到学员满意
- 重新生成时要参考学员给出的具体修改意见，不能只是换个说法
- 保持修改后的内容与未修改部分的风格一致
- 不使用 AskUserQuestion，所有交互通过纯文本完成
