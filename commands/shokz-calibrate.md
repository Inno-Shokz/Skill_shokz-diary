# shokz-calibrate

校准当天已生成日报，或校准 `.shokz/learner-profile.md` 中的动态画像观察。

## 交互模式

**纯对话模式**：不使用 AskUserQuestion。直接输出问题，等待学员自然回复。

## 校准范围

支持两类校准：

1. **日报正文校准**：修改正文中的措辞、结构、事实、来源标记、风险或明日计划。
2. **画像校准**：修改、删除、降级或标记过期的画像观察。

禁止反向改写历史原话。附录中的学员原始回答和 Agent 追问记录必须保真保留。

## 前置检查

1. 获取当前日期（YYYY-MM-DD）。
2. 读取 `.shokz/profile.json`；若不存在，提示先运行 `/shokz-diary:shokz-init`。
3. 从 profile 读取 `outputDir`；若无则默认 `.shokz/reports/`。
4. 读取 `{outputDir}/{YYYY-MM-DD}.md`。
5. 若今天没有日报，输出并终止：
   ```
   今天还没有生成日报。请先运行 /shokz-diary:shokz-diary
   ```
6. 读取 `.shokz/learner-profile.md`（若存在）。

## 执行流程

### Step 1: 选择校准对象

展示选项：

```markdown
你想校准哪一类内容？

1. 日报正文：修改今日推进、关键结论、卡点风险、明日计划、需导师确认等提交内容。
2. 学员画像：修改、删除、降级或标记过期动态画像观察。
3. 两者都改：先改日报正文，再根据结果调整画像。

请直接回复选项编号，并说明哪里不满意。
```

等待学员回复。

### Step 2A: 日报正文校准

若学员选择日报正文：

1. 展示当天日报正文区，不展示附录原始对话全文，除非需要核对来源。
2. 输出：
   ```markdown
   请说明要修改哪一部分：
   - 今日推进
   - 关键结论
   - 卡点与风险
   - 明日计划
   - 需导师确认
   - 管理视角补充
   - 问题沉淀候选
   - 其他

   也请说明原因：事实错误、太笼统、方向偏了、来源标记不对、需要更企业化、需要保守一点等。
   ```
3. 基于以下信息重写指定部分：
   - 原始对话附录
   - 事实清单确认记录
   - 学员修改意见
   - `.shokz/profile.json`
   - `.shokz/learner-profile.md`（若存在）
4. 只替换学员指定部分，未指定部分保持不变。
5. 不得改写附录 A 中的历史原话。

### Step 2B: 画像校准

若学员选择画像：

1. 展示 `.shokz/learner-profile.md` 中相关观察；若文件不存在，说明当前没有画像文件。
2. 输出：
   ```markdown
   请说明要如何处理画像观察：
   - 覆盖：用新的观察替换旧观察
   - 删除：移除不应保留的观察
   - 降级：从“已确认”降为“观察中”
   - 标记过期：保留历史，但不再作为当前判断依据
   - 补充证据：为观察增加证据来源
   ```
3. 根据学员指令更新画像。
4. 每条更新必须保留或补齐：
   - 观察内容
   - 证据来源
   - 首次出现日期
   - 最近验证日期
   - 状态：观察中 / 已确认 / 已修正 / 已过期

### Step 3: 更新日报中的校准记录

若修改了日报正文，在日报末尾追加或更新：

```markdown
---

## 附录 C：校准记录

| 时间 | 校准对象 | 修改说明 |
|------|----------|----------|
| {ISO 8601 时间} | {日报正文/画像} | {修改说明} |
```

若只修改画像，也在终端说明画像已更新，并建议下次日报读取该画像。

### Step 4: 保存并展示

1. 覆盖写入更新后的日报或画像文件。
2. 展示修改后的相关区块。
3. 输出：
   ```
   校准已完成。如仍需调整，可再次运行 /shokz-diary:shokz-calibrate
   ```

## 质量红线

- 不修改附录 A 中的学员原始回答。
- 不修改附录 A 中的 Agent 历史追问。
- 不把学员的修正意见伪装成当时原话。
- 不保留被学员否认的画像判断。
- 不把一次表现升级为永久标签。
- 不使用 AskUserQuestion。
